# pmerge

Браузерное расширение (Chrome, Manifest V3) с плавающим окном AI-ассистента. Окно можно перетаскивать по странице; из выходной точки («порта») можно тянуть связи к элементам страницы (картинки, ссылки, текст). По нажатию «Отправить» контекст (промт + метаданные связанных элементов) отправляется в OpenAI API; ответ показывается в окне ассистента.

## Сценарий использования

1. Открыть, например, Pinterest.
2. Появится плавающее окно «AI Assistant».
3. Ввести промт в поле (например: «Сделай картинку с таким вайбом на тему космического кафе»).
4. Тянуть от синей точки «Привязать к элементу» к понравившимся картинкам на странице — появятся связи (веревки).
5. Нажать «Отправить». Расширение отправит промт и метаданные связанных картинок (URL, alt, заголовки) в OpenAI; ответ отобразится в окне.
6. Клик по «веревке» удаляет связь.

## Установка

1. Клонировать или скачать репозиторий.
2. Установить зависимости и собрать:
   ```bash
   npm install
   npm run build
   ```
3. В Chrome: `chrome://extensions/` → «Режим разработчика» → «Загрузить распакованное расширение» → выбрать папку проекта (ту, где лежит `manifest.json`).
4. Открыть настройки расширения и ввести **API Key**:
   - **Groq** (по умолчанию, бесплатно): ключ с [console.groq.com](https://console.groq.com) — Llama 3.1 8B, ~14 400 запросов/день
   - **OpenAI** (платно): ключ с platform.openai.com

## Сборка

- `npm run build` — однократная сборка в папку `dist/`.
- `npm run watch` — сборка при изменении файлов.

## Структура проекта

- `manifest.json` — манифест расширения (MV3).
- `src/background/` — service worker: приём сообщений, вызов OpenAI API.
- `src/content/` — контент-скрипт: overlay (Shadow DOM), окно ассистента, порт, веревки (SVG), граф связей, извлечение метаданных и локаторов.
- `src/shared/` — общие типы и константы сообщений.
- `src/options/` — страница настроек (API key).
- `options.html` — HTML страницы настроек.

## Безопасность

- API ключ хранится только в `chrome.storage.local` и используется только в background. Не захардкожен и не логируется.
- Хранение ключа в расширении небезопасно для продакшена; для личного прототипа допустимо. В продакшене рекомендуется прокси-бэкенд с ключом на сервере.

## Разрешения

- `storage` — сохранение настроек и опционально позиции окна.
- `activeTab` — доступ к активной вкладке для контент-скрипта.
- `host_permissions`: `https://*.pinterest.com/*` и `<all_urls>` — для работы на любых страницах и на Pinterest.
